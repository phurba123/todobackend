Events to listen

userId event
This event ("userId") has to be listened to get the updates from another user.
Example output is as follows:
                            {
                                    message: 'xyz has edited your list',
                                }
Auth Error (optional)
This event ("auth-error-(your userId)") has to be listened to verify auth token.

Events to emit

verifyUser event
This event ("verifyUser") has to be emitted on the user's end to verify user authentication.
The following data has to be emitted
                           {
                                authToken:'jfskj983u9fj00f093jfsdi',
                                userId:'jifsd89f'
                                
                            }
notify-updates event
This event ("notify-updates") has to be emitted to share the update message from a user to friend
The following data has to be emitted
                           {
                                friendId:'djfi439fs,
                                message: 'xyz has edited your list',
                                friendEmail:'xyz@email.com',
                                friendName:'ggggg'
                                
                            }
